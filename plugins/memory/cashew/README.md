# Cashew Memory Provider Plugin

Persistent thought-graph memory for Hermes Agent, backed by [cashew-brain](https://github.com/rajkripal/cashew).

## Quick Start

```bash
hermes plugins install cashew
hermes memory setup    # configure provider
hermes gateway restart
```

## Configuration

Configured via `~/.hermes/cashew.json` (auto-generated by `hermes memory setup`).

### Required

| Key | Default | Description |
|-----|---------|-------------|
| `llm_aux_role` | `None` | Set to `"memory"` to enable LLM-powered extraction and thinking. Requires `auxiliary.memory` in Hermes `config.yaml`. |

### Performance

| Key | Default | Description |
|-----|---------|-------------|
| `sync_queue_timeout` | `30.0` | Seconds to wait for the sync queue to drain on shutdown |
| `think_interval` | `10` | How many sync turns between think cycles (0 = disable) |

### Features

| Key | Default | Description |
|-----|---------|-------------|
| `sleep_cycles` | `True` | Refactored sleep cycle (v0.8.0+) — fast ~4s consolidation via vectorized numpy, runs in on_session_end. |
| `think_cycles` | `True` | Enable periodic insight generation |

### Storage

| Key | Default | Description |
|-----|---------|-------------|
| `cashew_db_path` | `cashew/brain.db` | Relative to `$HERMES_HOME` |
| `embedding_model` | `thenlper/gte-large` | Sentence-transformers model for embeddings (1024-dim) |

### Retrieval

| Key | Default | Description |
|-----|---------|-------------|
| `recall_k` | `5` | Context fragments returned per query |

Legacy tuning keys from older releases remain readable for compatibility, but
are not advertised because cashew-brain 1.x has no supported API to consume
them. Non-default legacy values emit a startup warning.

## Tools

The provider exposes two tools to the LLM:

### `cashew_query`

Recall context from the thought graph. Accepts `query` (required), `max_nodes`, `domain`, `tag`, `exclude_tags`.

### `cashew_extract`

Explicitly persist a conversation turn. Accepts `user_content` and `assistant_content`. Bypasses the sync queue — runs synchronously.

## LLM Integration

When `llm_aux_role` is set to `"memory"`, the provider reads the `auxiliary.memory` section from Hermes `config.yaml`:

```yaml
auxiliary:
  memory:
    provider: opencode-go
    model: deepseek-v4-flash
    base_url: https://opencode.ai/zen/go/v1
```

This enables:
- **LLM extraction** — Cashew uses the model to extract typed nodes and edges from conversation turns
- **Think cycles** — Periodic cross-domain insight generation (every `think_interval` turns)
- **Sleep cycles** — Graph consolidation (dedup, cross-link, dream, decay, core memory promotion) on session end
- **Pre-compress insight extraction** — Before context compression discards old messages, extracts conversation-arc patterns (topic shifts, framing changes, implicit decisions) using a dedicated LLM prompt. Silent-degrades without LLM.

Without LLM, Cashew falls back to heuristic extraction.

## Logs

Plugin logs go to `~/.hermes/logs/agent.log` (not gateway.log — the gateway has a component filter).

Key log lines to watch:
```
INFO plugins.memory.cashew: llm_aux_role='memory': using opencode-go deepseek-v4-flash ...
INFO plugins.memory.cashew: on_session_end called: ...
INFO plugins.memory.cashew: think cycle produced 2 insight(s) on cluster: ...
WARNING plugins.memory.cashew: think cycle failed ...
WARNING plugins.memory.cashew: on_pre_compress failed ...
```
